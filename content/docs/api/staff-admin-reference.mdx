The Admin API provides staff and administrators with access to user management, moderation, billing, product configuration, and system monitoring tools.

## Authentication

### Admin Token

All admin endpoints require either:
1. **Admin session** (via browser login with admin role)
2. **Admin API key** (for backend services)

Admin users have `role: "ADMIN"` or `role: "SUPERADMIN"` in the database.

```bash
curl -H "Authorization: Bearer ADMIN_API_KEY" \
  https://embrly.ca/api/admin/users
```

---

## Authorization Levels

| Level | Description | Can Access |
|-------|-------------|-----------|
| USER | Regular user | Only own data via user API |
| ADMIN | Staff/moderator | Moderation, reports, basic user data |
| SUPERADMIN | Owner/lead admin | Everything, including system config |

## User Management

### List All Users

**GET `/api/admin/users`**

List all users with filtering and search.

**Authentication:** ADMIN+

**Query Parameters:**
- `page` (default: 1) тАФ Page number
- `limit` (default: 50) тАФ Users per page
- `search` тАФ Search by name, email, or urlId
- `role` тАФ Filter by `USER`, `ADMIN`, `SUPERADMIN`
- `status` тАФ Filter by `active`, `banned`, `suspended`
- `createdAfter` тАФ ISO date (created after this date)
- `sort` тАФ `recent`, `active`, `storage-usage`

**Response (200):**

### Ban/Suspend User

**POST `/api/admin/users/[userId]/ban`**

Ban or suspend a user account.

**Authentication:** ADMIN+

**Request Body:**
```json
{
  "type": "temporary", // or "permanent"
  "durationDays": 30,  // Only for temporary
  "reason": "Violation of terms of service",
  "notifyUser": true
}
```

**Response (200):**
```json
{
  "success": true,
  "data": {
    "userId": "user_123",
    "bannedAt": "2026-03-29T15:30:00Z",
    "banType": "temporary",
    "banExpiresAt": "2026-04-28T15:30:00Z",
    "banReason": "Violation of terms of service",
    "reversible": true
  }
}
```

## Error Handling

Admin endpoints follow the same error format as user endpoints:

```json
{
  "success": false,
  "error": "Permission denied",
  "code": "ADMIN_REQUIRED"
}
```

**Admin-Specific Error Codes:**

| Code | Meaning |
|------|---------|
| `ADMIN_REQUIRED` | User must be ADMIN+ |
| `SUPERADMIN_REQUIRED` | User must be SUPERADMIN |
| `INVALID_USER_ID` | User not found |
| `CANNOT_MODIFY_SELF` | Can't modify own account |
| `INVALID_ACTION` | Action not allowed |

## Best Practices

1. **Always log actions** тАФ Document why you took action
2. **Use dry-run first** тАФ Preview bulk operations before executing
3. **Require 2FA** тАФ All admin accounts should have 2FA enabled
4. **Monitor audit logs** тАФ Regularly review admin activity
5. **Least privilege** тАФ Don't promote users to SUPERADMIN unless necessary
6. **Document policies** тАФ Keep clear rules for moderation decisions
